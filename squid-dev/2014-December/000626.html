Current 3.5 branch is apparently proving to be very stable
operationally. Although its 2 months behind the original plan.

On the plus side it does mean I was able to push in the changes needed
for HTTP/2 forward-compatibility so Squid should deal with the future
HTTP environment better than expected.

* Christos has one major bug fix to go in for Peek-n-Splice. Which is
big enough behavour change to mean one more beta and will reset the
stable countdown again.

* I'm inclined to accept Markus performance patch for kerberos helpers
as well since we have another round anyway and the gains are desirable.

* Plus a bunch of documentation updates needed to complete the
copyright boilerplate project.


If no other major bugs come up we should have a stable early or mid
January.


I am aware of big issues in the areas of memory pseudo-leaks (failure
to de-reference/release unneeded memory), performance (100% CPU or
hang), auth behaviour and Windows build support. Most appear to be
artifacts of Squids current state in the C->C++ transition. All are
inherited from earlier series.
 The fix for those (even finding them) is not going to be easy and
will continue over the 3.5 series lifetime.

Amos
