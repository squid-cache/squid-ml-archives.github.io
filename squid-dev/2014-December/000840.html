I noticed that some products do use a session ID and I am not sure
about the plans inside squid.
Until now I noticed the session ID in reverse proxies error pages and
on some logging output.

If there are any plans to implement a session\connection ID one
question rises:
Are we going to use a connection ID? or a session ID? request ID? others?
What scheme will be used for the ID?
What is the maximum concurrent connections we expect a proxy to be
able to handle?

Most of the above decisions would probably also need to take in
account HTTP/2.0.

If anyone have any ideas or thoughts about it please open them.
