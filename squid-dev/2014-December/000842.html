On 12/29/2014 12:20 AM, Amos Jeffries wrote:
> What "session ID"? There is no such thing as sessions in HTTP.

There is no such thing as a session ID in HTTP you are right I was
talking about something else.
My basic idea of a session or connection ID what ever that should be
talked about is mostly for debugging\tracking purposes.
Sometimes it's very tricky to track inside the debugging logs what
line goes with what request and\or response.

I have seen that here and there the usage of a session ID is being
used with a forward proxy to identify a specific proxy connection that
in many cases correspond with a auth user.
For example 1 incoming TCP connection has an ID and there for can be
found in the logs.
The idea I have seen used is for logging purposes instead of debugging.

mailing list.
