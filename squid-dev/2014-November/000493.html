First of all does not do a full rebuild.
It is just rebuilding the "size" file of the database.
Sometimes when an ssl_crtd daemon crash may leave this file empty.
This patch fixes ssl_crtd to do the following:
    - on ssl_crtd startup checks the "size" file and if it is not have a 
valid value, rebuild this file.
   - Every time ssl_crtd daemon, needs to update "size" file checks if 
it has a valid value. If not rebuild it.
