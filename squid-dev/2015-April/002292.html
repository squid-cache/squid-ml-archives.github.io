Ref bug 3608:

Squid 3.5 now has a -n option to set the service name so you can run 
multiple instances concurrently.  This is used in ipc/Port.cc but not 
ipc/mem/Segment.cc.  Consequently, if you're running multiple Squids, 
they all try to use the same name for shm_open().

Bug 3608 included a patch to resolve this, but it looks like it was 
never applied to Squid 3.5.
