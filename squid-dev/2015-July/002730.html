Hello,

    The attached patch finishes work started by Amos in 2011:
http://www.squid-cache.org/mail-archive/squid-dev/201112/0080.html

We already support %un (any available user name, without triggering
authentication) for access logging. There is an apparently a persistent
need for it when talking to external ACL helpers. Adding %un support for
those helpers also matches the overall intent of supporting virtually
all logformat codes there, unifying the two interfaces.

The patch updates/details existing %un documentation as well.

An earlier version of this patch was production tested.

Amos, if you are OK with this patch, then please commit it yourself so
that we do not have to come up with a long list of Authors. You started
this work and having no other author explicitly listed is perfectly fine
with us.


HTH,

Alex.
