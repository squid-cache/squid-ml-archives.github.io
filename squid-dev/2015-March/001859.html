At our recent meeting the Foundation board had a bit of impromptu
discussion about the ongoing saga of C++11 features in trunk versus OS
distribution compiler support in the long-term.

As a result we concluded it was a good idea to (try to) retain support
for the default compilers available in the LTS distro versions last LTS
release provided it was 6-months or older. If the LTS was less than 6
months we would still be looking at the one past it.

Along those grounds we are now in a position to decide about CentOS 6.x
support in trunk.

* CentOS 7 latest was made available in July 2014, and now 6-months old.

* This probably means RHEL 6 support is in an identical situation,
though we dont test for that OS.

* I'm currently holding back some useful upgrades to security and code
simplicity due solely to the age of that compiler.

* The trunk code is not expected to be released formally for some months.


So, how much push back is there on dropping CentOS 6 from the trunk
stability testing soonish?
