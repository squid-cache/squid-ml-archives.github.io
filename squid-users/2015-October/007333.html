On 27/10/2015 11:54 p.m., Tarik Demirci wrote:
> Hello,
> Is there any plan to add support for websocket when using intercept mode?
> 
> Currently, I use SslPeekAndSplice feature but this brokes many
> websites using websocket (one example is web.whatsapp.com). As a
> workaround, after peeking at step 1, splicing problematic sites and
> bumping the rest works. But maintaining this list is tiring and I
> can't use content filtering for these sites. It would be much better
> if squid had support for websocket.
> 
> Related issue:
> http://bugs.squid-cache.org/show_bug.cgi?id=4349
> 

As you see it has been requested and everyone is just waiting for
somebody else to do it for them for free.
<http://wiki.squid-cache.org/SquidFaq/AboutSquid#How_to_add_a_new_Squid_feature.2C_enhance.2C_of_fix_something.3F>

To be frank, the upstream developers are more interested in getting
Squid to be more performant and getting HTTP/2 working. We have very
little interest in making Squid become just a slow blind packet relay
for traffic that would be better passed over TCP connections. Which is
what WebSockets is designed to lead to. So the hobbyists are not going
to do it for free any time soon.
