<p class="MsoNormal">Both instances are configured with the following options:<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal" style="margin-left:.5in">request_header_access User-Agent deny all<o:p></o:p></p>
<p class="MsoNormal" style="margin-left:.5in">request_header_replace User-Agent someagent; (squid proxy header rewrite DID-IT)<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">The Fedora instance running v3.3.13 works just fine and rewrites headers like a champ.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">The RHEL instance running v3.1.10 fails when I start squid with the following message:<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">2014/11/17 17:28:18| cache_cf.cc(364) parseOneConfigFile: squid.conf:61 unrecognized: 'request_header_replace'<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">I suspected that it was because I am missing the –enable-http-violations option.  However, neither instance displays --enable-http-violations or –disable-http-violations when I run “squid –v”.  Moreover, the squid.spec files for both RPMs
 did not include any reference to –enable-http-violation or –disable-http-violation.  Is “–enable-http-violations” the default config option?<o:p></o:p></p>
