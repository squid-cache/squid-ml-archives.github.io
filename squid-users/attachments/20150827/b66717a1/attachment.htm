</span>The server connection got disconnected between sending Squid the reply<br>
headers and the message payload they were attached to.<br>
<br>
If you have a Squid between 3.2.0 and 3.5.5 (inclusive) please upgrade.<br>
Which is processing SSL-bump, NTLM or Negotiate auth (even just relaying<br>
those in www-auth form). It is probably bug 3329 related.<br>
<br>
If you have a more current Squid "debug_options 11,2" should show you<br>
the HTTP headers going through to eyeball if they had any kind of fatal<br>
syntax problem that would make Squid abandon the connection.<br>
<br>
Otherwise it would seem to be the server disconnecting. There can be a<br>
lot of reasons for that.<br>
