<p dir="ltr">I'd think a regex consumes a lot more resources than server name, but don't know if it is significant. <br>
Anyway, without more details we can't be sure the server name not matching is the problem. </p>
<p dir="ltr">We need access logs and client(browser) details. </p>
<p dir="ltr">By the way, acl excludeSSL ssl::server_name <a href="http://web.whatsapp.com">web.whatsapp.com</a><br>
Would not work, whatsapp uses some subdomains that also should not be bumped. </p>
