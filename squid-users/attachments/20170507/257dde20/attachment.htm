        <p>I am using squid on Windows (<a

            moz-do-not-send="true">http://squid.diladele.com/</a>), but,
          they don't compile it with support to 'intercept' option.</p>
        <p><br>
        </p>
        <p>Is that possible to enable it for Windows on current version?</p>
        <p><br>
        </p>
        <p>I am using a very old version 2.7STABLE 8 with support
          'transparent' option on Windows, available here:
          <a
            href="http://squid.acmeconsulting.it/download/dl-squid.html"
            class="OWAAutoLink" id="LPlnk881445" previewremoved="true"
            moz-do-not-send="true">
            http://squid.acmeconsulting.it/download/dl-squid.html</a>
          (maybe someone who understand how it works, can port these
          option from the old system to the new system, I don't know
          exactly...).<br>
        </p>
        <p><br>
        </p>
        <p>Based on Squid documentation there is these options, but they
          dont apply to Windows:</p>
        <p><br>
        </p>
        <p>- <a
href="http://wiki.squid-cache.org/SquidFaq/InterceptionProxy#Concepts_of_Interception_Caching"
            class="OWAAutoLink" id="LPlnk837340" previewremoved="true"
            moz-do-not-send="true">
http://wiki.squid-cache.org/SquidFaq/InterceptionProxy#Concepts_of_Interception_Caching</a>
          :<br>
        </p>
        <ul>
          <li>
            <p class="line862">For Linux configure Squid with the <tt>--enable-linux-netfilter</tt>
              option.
              <span class="anchor" id="line-69"></span></p>
          </li>
          <li>
            <p class="line862">For *BSD-based systems with IP filter
              configure Squid with the
              <tt>--enable-ipf-transparent</tt> option. <span
                class="anchor" id="line-70"></span></p>
          </li>
          <li>
            <p class="line862">If you're using OpenBSD's PF configure
              Squid with <tt>--enable-pf-transparent</tt></p>
          </li>
        </ul>
