<p class="MsoNormal">Greetings all,<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">The version 5.2.0.210A of Web Safety ICAP web filter for Squid proxy is announced as Release Candidate.
<o:p></o:p></p>
<p class="MsoNormal">It is now ready for broad deployment to production systems. The code is complete and anything new will only be added to the upcoming 5.3.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">This version contains the following fixes and improvements:<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoListParagraph" style="margin-left:45.35pt;text-indent:-.25in;mso-list:l0 level1 lfo1">
<![if !supportLists]><span style="font-family:Symbol"><span style="mso-list:Ignore">·<span style="font:7.0pt "Times New Roman"">       
</span></span></span><![endif]>Added management sections for Squid cache (refresh patterns) and logging submodules to Admin UI. It is now very easy to enable different log level for a separate Squid module to make troubleshooting simpler.<o:p></o:p></p>
<p class="MsoListParagraph" style="margin-left:45.35pt;text-indent:-.25in;mso-list:l0 level1 lfo1">
<![if !supportLists]><span style="font-family:Symbol"><span style="mso-list:Ignore">·<span style="font:7.0pt "Times New Roman"">       
</span></span></span><![endif]>New version of definition files database. Some categories were combined to make the usage more straightforward, several new categories added.<o:p></o:p></p>
<p class="MsoListParagraph" style="margin-left:45.35pt;text-indent:-.25in;mso-list:l0 level1 lfo1">
<![if !supportLists]><span style="font-family:Symbol"><span style="mso-list:Ignore">·<span style="font:7.0pt "Times New Roman"">       
</span></span></span><![endif]>Added support for “brotli” transfer encoding, greatly improving filtering on YouTube and other Google services.<o:p></o:p></p>
<p class="MsoListParagraph" style="margin-left:45.35pt;text-indent:-.25in;mso-list:l0 level1 lfo1">
<![if !supportLists]><span style="font-family:Symbol"><span style="mso-list:Ignore">·<span style="font:7.0pt "Times New Roman"">       
</span></span></span><![endif]>Improved correctness of traffic monitoring reports built over the Squid access logs.<o:p></o:p></p>
<p class="MsoListParagraph" style="margin-left:45.35pt;text-indent:-.25in;mso-list:l0 level1 lfo1">
<![if !supportLists]><span style="font-family:Symbol"><span style="mso-list:Ignore">·<span style="font:7.0pt "Times New Roman"">       
</span></span></span><![endif]>Added support for recently released pfSense 2.4. Also dropped support for FreeBSD 10 and added FreeBSD 11.<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">The version is available from <a href="https://www.diladele.com/download_next_version.html">
https://www.diladele.com/download_next_version.html</a> . It is recommended to use Ubuntu 16 and CentOS 7 based virtual appliances in production. Direct links to virtual appliances are:<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>

<p class="MsoNormal">Final release is expected in two weeks (approx. 15 of November, 2017).<o:p></o:p></p>
<p class="MsoNormal"><o:p> </o:p></p>
<p class="MsoNormal">Next version 5.3 will contain eCAP based ClamAV adapter antivirus from Measurement Factory, available from
<a href="http://www.e-cap.org/downloads">http://www.e-cap.org/downloads</a>  (except for FreeBSD 11), support for haproxy’s PROXY protocol and proxy pseudo authentication based on IP to Active Directory open source project Active Directory Inspector ( see
<a href="https://github.com/diladele/active-directory-inspector">https://github.com/diladele/active-directory-inspector</a> ).  Builds of 5.3 for now is in beta stage and thus freely available to Early Adopters community (
<a href="https://www.diladele.com/community.html">https://www.diladele.com/community.html</a> ).<o:p></o:p></p>
