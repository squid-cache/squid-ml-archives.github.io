<div dir="ltr"><div>There are significant differences between the Fedora->RHEL and Debian->Ubuntu relations.<br>Fedora is development oriented with short lifecycle and without support of previous versions. RHEL is built on Fedora with stability and enterprise needs in mind (they always ).<br><br>Debian provides more release stages<br> - stable -> Ubuntu TLS releases<br> - testing -> standard Ubuntu releases<br> - unstable<br><br>Ubuntu LTS releases are based on current Debian stable release. Standard Ubuntu relases are based on Debian testing. Both developer communities share parts of infra and are working "together".<br>More information can be read here <a href="https://www.ubuntu.com/community/debian">https://www.ubuntu.com/community/debian</a></div><div><br>
</div><div>This is major difference.</div><div><br></div><div>The RHEL->CentOS relation is of different type. CentOS is RHEL without copyrighted materials. Updates are usually released with delays. There are some other repositories which extend/modify CentOS core <a href="https://en.wikipedia.org/wiki/CentOS#REPOS">https://en.wikipedia.org/wiki/CentOS#REPOS</a><br></div><div><br></div><div>-- <br></div>Peter<br></div><div class="gmail_extra"><br><div class="gmail_quote">On Mon, Jun 4, 2018 at 10:38 AM, Amos Jeffries <span dir="ltr"><<a href="mailto:squid3@treenet.co.nz" target="_blank">squid3@treenet.co.nz</a>></span> wrote:<br><blockquote class="gmail_quote" style="margin:0 0 0 .8ex;border-left:1px #ccc solid;padding-left:1ex"><span class="">On 04/06/18 19:51, Eliezer Croitoru wrote:<br>
