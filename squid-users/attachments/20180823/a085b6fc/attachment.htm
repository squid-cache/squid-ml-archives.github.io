Nothing particular happened.<br>
<br>
HTTPS is sent differently through proxies. Using CONNECT requests with<br>
encrypted content. More details on that can be found at<br>
<<a href="https://wiki.squid-cache.org/Features/HTTPS" rel="noreferrer" target="_blank">https://wiki.squid-cache.org/Features/HTTPS</a>>.<br>
<br>
Browsers in particular refuse to display error messages sent in response<br>
to CONNECT requests. I think you will find the Squid does send the error<br>
page but the Browser is refusing to display it.<br>
<br>
<br>
The only thing you can do about this sad situation is use SSL-Bump<br>
feature on the encrypted traffic so Squid can send the error message<br>
within an encrypted response - which the Browser does display *if* it<br>
trusts the proxy TLS certificate used by SSL-Bump.<br>
<br>
Please be aware:<br>
  SSL-Bump feature is sometimes forbidden or restricted-use by law. So<br>
please get proper legal advice before using it. Anyone helping you setup<br>
the feature will be assuming that you have done that due diligence check.<br>
