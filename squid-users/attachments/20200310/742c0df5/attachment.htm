<p class="MsoNormal"><span lang="EN-US">Web Safety 7.3 – ICAP web filter for Squid proxy and Admin UI for Squid Proxy is now available for production use. The following changes and improvements are included into this build.<o:p></o:p></span></p>
<p class="MsoNormal"><span lang="EN-US"><o:p> </o:p></span></p>
<ul style="margin-top:0cm" type="disc">

 before.<o:p></o:p></span></li><li class="MsoListParagraph" style="margin-left:0cm;mso-list:l1 level1 lfo1"><span lang="EN-US">Added a setting to allow automatic download of missing intermediate HTTPS certificates. Disabled by default.<o:p></o:p></span></li><li class="MsoListParagraph" style="margin-left:0cm;mso-list:l1 level1 lfo1"><span lang="EN-US">Fixed some minor issues in report generation, added more debug output to the report log allowing for better understanding how much time was spent during reporting.<o:p></o:p></span></li><li class="MsoListParagraph" style="margin-left:0cm;mso-list:l1 level1 lfo1"><span lang="EN-US">Added support for Squid 4.10 on Ubuntu 18/Debian 10.<o:p></o:p></span></li><li class="MsoListParagraph" style="margin-left:0cm;mso-list:l1 level1 lfo1"><span lang="EN-US">Improved cluster configuration sync. The client nodes are able to upload the Squid’s access logs to the server node. The report building process at server node can
 now create integrated traffic history for the whole cluster.<o:p></o:p></span></li><li class="MsoListParagraph" style="margin-left:0cm;mso-list:l1 level1 lfo1"><span lang="EN-US">It is now possible to completely disable traffic monitoring and reporting from the Admin UI.<o:p></o:p></span></li></ul>
