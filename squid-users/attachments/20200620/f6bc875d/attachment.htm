of the Squid-5.0.3 beta release!<br>
<br>
<br>
This release is a security and feature update release resolving<br>
several issues found in the prior Squid releases.<br>
<br>
<br>
The major changes to be aware of:<br>
<br>
 * SQUID-2020:5 Denial of Service when using SMP cache<br>
   (CVE-2020-14059)<br>
<br>
This problem may allow a remote client to trigger a Squid worker<br>
assertion.<br>
<br>
This attack is limited to SMP Squids using shared memory cache<br>
and/or an SMP rock disk cache.<br>
<br>
<br>
See the advisory for patches:<br>
 <<a href="http://www.squid-cache.org/Advisories/SQUID-2020_5.txt" rel="noreferrer" target="_blank">http://www.squid-cache.org/Advisories/SQUID-2020_5.txt</a>><br>
<br>
<br>
 * SQUID-2020:6 Denial of Service issue in TLS handshake<br>
   (CVE-2020-14058)<br>
<br>
This problem allows a trusted client to perform Denial of Service<br>
when opening TLS connections with a server for HTTPS.<br>
<br>
This problem allows a trusted client to perform Denial of Service<br>
when opening TLS connections to a server for SSL-Bump intercepted<br>
transactions.<br>
<br>
This attack is limited to Squid built with OpenSSL features and<br>
opening peer or server connections for HTTPS traffic and SSL-Bump<br>
server handshakes.<br>
<br>
See the advisory for patches:<br>
 <<a href="http://www.squid-cache.org/Advisories/SQUID-2020_6.txt" rel="noreferrer" target="_blank">http://www.squid-cache.org/Advisories/SQUID-2020_6.txt</a>><br>
<br>
<br>
 * Happy Eyeballs: Do not discard viable reforwarding destinations<br>
<br>
When Happ Eyeballs starts opening two connections, both destinations<br>
are removed from the destinations list. As soon as one connection<br>
(X) succeeded, the other destination (Y) was essentially forgotten. If<br>
Squid, after using X, decided to reforward the request, then the request<br>
was never reforwarded to Y. We now return Y to the list of possible<br>
destinations.<br>
<br>
<br>
<br>
  All users of Squid-5 are urged to upgrade as soon as possible.<br>
<br>
  All users of Squid-4 and older are encouraged to plan for upgrade.<br>
<br>
