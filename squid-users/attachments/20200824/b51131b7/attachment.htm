<blockquote style="border:none;border-left:solid #CCCCCC 1.0pt;padding:0in 0in 0in 6.0pt;margin-left:4.8pt;margin-right:0in">
<p class="MsoNormal">The Squid HTTP Proxy team is very pleased to announce the availability<br>
of the Squid-4.13 release!<br>
<br>
<br>
This release is a security release resolving several issues found in<br>
the prior Squid releases.<br>
<br>
<br>
The major changes to be aware of:<br>
<br>
 * SQUID-2020:8 HTTP(S) Request Splitting<br>
   (CVE-2020-15811)<br>
<br>
This problem is serious because it allows any client, including<br>
browser scripts, to bypass local security and poison the browser<br>
cache and any downstream caches with content from an arbitrary<br>
source.<br>
<br>
See the advisory for patches:<br>
 <<a href="https://github.com/squid-cache/squid/security/advisories/GHSA-c7p8-xqhm-49wv" target="_blank">https://github.com/squid-cache/squid/security/advisories/GHSA-c7p8-xqhm-49wv</a>><br>
<br>
<br>
 * SQUID-2020:9 Denial of Service processing Cache Digest Response<br>
   (CVE pending allocation)<br>
<br>
This problem allows a trusted peer to deliver to perform Denial<br>
of Service by consuming all available CPU cycles on the machine<br>
running Squid when handling a crafted Cache Digest response<br>
message.<br>
<br>
This attack is limited to Squid using cache_peer with cache<br>
digests feature.<br>
<br>
See the advisory for patches:<br>
 <<a href="https://github.com/squid-cache/squid/security/advisories/GHSA-vvj7-xjgq-g2jg" target="_blank">https://github.com/squid-cache/squid/security/advisories/GHSA-vvj7-xjgq-g2jg</a>><br>
<br>
<br>
 * SQUID-2020:10 HTTP(S) Request Smuggling<br>
   (CVE-2020-15810)<br>
<br>
This problem is serious because it allows any client, including<br>
browser scripts, to bypass local security and poison the proxy<br>
cache and any downstream caches with content from an arbitrary<br>
source.<br>
<br>
<br>
See the advisory for patches:<br>
 <<a href="https://github.com/squid-cache/squid/security/advisories/GHSA-3365-q9qx-f98m" target="_blank">https://github.com/squid-cache/squid/security/advisories/GHSA-3365-q9qx-f98m</a>><br>
<br>
<br>
 * Bug 5051: Some collapsed revalidation responses never expire<br>
<br>
This bug appears as a 4xx or 5xx status response becoming the only<br>
response delivered by Squid to a URL when Collapsed Forwarding<br>
feature is used.<br>
<br>
It primarily affects Squid which are caching the 4xx/5xx status<br>
object since Bug 5030 fix in Squid-4.11. But may have been<br>
occurring for short times on any proxy with Collapsed Forwarding.<br>
<br>
<br>
<br>
 * SSL-Bump: Support parsing GREASEd (and future) TLS handshakes<br>
<br>
Chrome Browser intentionally sends random garbage values in the<br>
TLS handshake to force TLS implementations to cope with future TLS<br>
extensions cleanly. The changes in Squid-4.12 to disable TLS/1.3<br>
caused our parser to be extra strict and reject this TLS garbage.<br>
<br>
This release adds explicit support for Chrome, or any other TLS<br>
agent performing these "GREASE" behaviours.<br>
<br>
<br>
 * Honor on_unsupported_protocol for intercepted https_port<br>
<br>
This behaviour was one of the intended use-cases for unsupported<br>
protocol handling, but somehow was not enabled earlier.<br>
<br>
Squid should now be able to perform the on_unsupported_protocol<br>
selected action for any traffic handled by SSL-Bump.<br>
<br>
<br>
  All users of Squid are urged to upgrade as soon as possible.<br>
<br>
<br>
See the ChangeLog for the full list of changes in this and earlier<br>
releases.<br>
