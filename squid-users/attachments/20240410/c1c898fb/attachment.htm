<br>
Best way to find out is to run "squid -k parse", which should be done <br>
after upgrades as well to identify and fix changes between versions as <br>

<br>
<br>
> I actually get allot of hits and it functions amazing, so I wanted to <br>
> share this in case I could improve something. Is there any issues with <br>
> security?<br>
<br>
Yes, the obvious one is "DONT_VERIFY_PEER" disabling TLS security <br>
entirely on outbound connections. That particular option will prevent <br>
you even being told about suspicious activity regarding TLS.<br>
<br>
Also there are a few weird things in your TLS cipher settings, such as <br>
this sequence " EECDH+aRSA+RC4:...:!RC4 "<br>
Which as I understand, enables the EECDH with RC4 hash, but also <br>
forbids all uses of RC4.<br>
<br>
<br>
> I am concerned that an invasive container could become <br>
> installed in the cache and data marshal the network card.<br>
> <br>
<br>
You have a limit of 4 MB for objects allowed to pass through this proxy, <br>
exception being objects from domains listed in the "windowsupdate" ACL <br>
(not all Windows related) which are allowed up to 512 MB.<br>
<br>
For the general case, any type of file which can store an image of some <br>
system is a risk for that type of vulnerability can be cached.<br>
<br>
The place to fix that vulnerability properly is not the cache or Squid. <br>
It is the OS permissions allowing non-Squid software access to the cache <br>
files and/or directory.<br>

> # Match against the current step during ssl_bump evaluation [fast]<br>
> # Never matches and should not be used outside the ssl_bump context.<br>
> #<br>
> # At each SslBump step, Squid evaluates ssl_bump directives to find<br>
> # the next bumping action (e.g., peek or splice). Valid SslBump step<br>
> # values and the corresponding ssl_bump evaluation moments are:<br>
> # SslBump1: After getting TCP-level and HTTP CONNECT info.<br>
> # SslBump2: After getting TLS Client Hello info.<br>
> # SslBump3: After getting TLS Server Hello info.<br>
> # These ACLs exist even when 'SSL/MITM Mode' is set to 'Custom' so that<br>
> # they can be used there for custom configuration.<br>
